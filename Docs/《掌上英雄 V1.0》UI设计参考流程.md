## 🛠️ AI + Figma 实战工作流：从线稿到代码交付

### 第一阶段：输入准备 (The "Double Reference" Method)

为了确保 AI 严格遵守布局和风格，您需要准备两类核心素材：

1. **布局参考（Structure Ref）：** 您的黑白线稿或灰盒草图。AI 将提取其中的组件位置、比例和层级。
2. **视觉参考（Style Ref）：** 您看中的风格图。AI 将提取其中的材质（如卡牌质感）、配色（如品牌色）和光影。
3. **提示词词典（Prompt Dictionary）：** 将您的功能需求翻译成 AI 易读的 Markdown 列表。

---

### 第二阶段：AI 生成规范 (Markdown Prompt 模板)

将以下模板直接输入给支持图像生成的 AI（如 GPT-Image-2、DALL-E 3 或 SD）。**注意：** 必须使用 Markdown 结构，因为它能强制 AI “分块”思考。

```markdown
# 任务：根据参考图生成 [游戏/产品名称] 界面 UI
# 核心约束
- 布局参考：严格遵循附件 1 (线稿) 的元素位置。
- 风格参考：严格遵循附件 2 (参考图) 的配色 (#Hex) 与材质。

# 页面结构 (Page Structure)
- [区域 1: 如顶部 HUD]: 包含 [元素 A]、[元素 B]，左对齐。
- [区域 2: 中心内容]: 包含 [元素 C: 卡牌槽位]，居中分布。
- [区域 3: 底部交互]: 包含 [元素 D: 丢弃按钮]，右下角。

# 视觉参数 (Visual Specs)
- 主色调: #XXXXXX (从风格图中提取)
- 圆角规范: 12px / 16px
- 阴影层级: 柔和阴影 (y:4, blur:12, rgba(0,0,0,0.1))
- 字体要求: [字体名称]，文字必须清晰可读，禁止乱码。

# 输出要求
- 视角: 纯正交正视角 (Orthographic View)，禁止 3D 透视。
- 背景: 纯色背景或透明通道 (如支持)，方便后期切图。

```

### 第三阶段：迭代反馈 (像素级修正)

如果 AI 输出的第一版有偏差，不要说“感觉不对”，要像给程序员下 Bug 单一样给指令：

| 偏差类型 | 错误示范 | **正确指令 (AI 更听话)** |
| --- | --- | --- |
| **位置偏差** | 按钮太靠上了 | 请将底部按钮的 Y 轴坐标向下移动，增加与卡牌区的间距 24px。 |
| **风格偏差** | 颜色不够高级 | 请降低主色 #XXXXXX 的饱和度，并增加金属拉丝纹理。 |
| **文字错误** | 文字是乱码 | 请将中间的标题文字明确修改为 "Inventory"，保持加粗。 |

---

### 第四阶段：开发交付物 (Output For Dev)

AI 生成的图是 PNG，不能直接用。您需要按照以下流程输出给实际开发：

#### 1. 资产切图 (Assets)

在 Figma 中利用生成的 AI 图作为底图，快速绘制矢量组件。

* **导出格式：** `.png` (2x/3x)，带透明通道。
* **九宫格 (9-Slice)：** 标注出 UI 底板的可拉伸区域。

#### 2. 布局配置 (Layout Config)

由于 AI 图已确定了布局，您可以直接输出一份 JSON 供程序读取坐标：

```json
{
  "screen": "Main_Game_UI",
  "elements": {
    "card_slot": { "pos": [960, 540], "size": [200, 300] },
    "discard_btn": { "pos": [1700, 900], "size": [80, 80], "asset": "btn_x_red.png" }
  }
}

```

#### 3. 交互说明文档 (Interaction MD)

使用 Markdown 编写界面逻辑，确保开发理解 AI 图中的按钮是怎么“动”的：

> ### 交互逻辑：手动弃牌
> 
> 
> * **触发条件：** 选中 [卡牌项目] 后，底部 X 按钮高亮。
> * **视觉反馈：** 点击 X 按钮，卡牌执行 `FadeOut` 动画，播放 `discard.mp3`。
> * **状态变化：** 对应的 JSON 槽位数据清空。
> 
>
