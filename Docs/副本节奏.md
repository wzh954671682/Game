## 一、JSON 标准模板

### 1.1 monster_templates.json — 怪物基础数值表

路径: `Data/monster_templates.json`

```json
{
  "monster_templates": {
    "monster_01": {
      "base_hp": 100,
      "base_speed": 100.0,
      "base_wall_damage": 1,
      "base_attack_damage": 5
    },
    "monster_02": {
      "base_hp": 300,
      "base_speed": 60.0,
      "base_wall_damage": 3,
      "base_attack_damage": 10
    },
    "monster_03": {
      "base_hp": 800,
      "base_speed": 40.0,
      "base_wall_damage": 5,
      "base_attack_damage": 20
    }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `monster_templates` | Object | 顶层容器，key 为 `monster_id` |
| `base_hp` | int | 基础生命值，实战中乘以波次的 `hp_multiplier` |
| `base_speed` | float | 每帧下移像素 (px/s) |
| `base_wall_damage` | int | 碰到城墙底线时造成的伤害 |
| `base_attack_damage` | int | 与英雄对砍时每次攻击的伤害 |

> **扩展规则**: 新增怪物类型时，在此文件追加新的 `monster_XX` 条目即可，`WaveDirector` 会自动读取。

| `grid.tile_count` | int | 地块总数 (当前固定 25) |

### 1.3 level_stage_config.json — 关卡波次脚本表

路径: `Data/level_stage_config.json`

```json
{
  "stages": {
    "stage_001": {
      "map_id": 1,
      "waves": [
        {"spawn_time": 0.0,  "monster_id": "monster_01", "lane": 2, "hp_multiplier": 1.0},
        {"spawn_time": 6.0,  "monster_id": "monster_01", "lane": 0, "hp_multiplier": 1.0},
        {"spawn_time": 12.0, "monster_id": "monster_01", "lane": 4, "hp_multiplier": 1.0},
        {"spawn_time": 20.0, "monster_id": "monster_02", "lane": 2, "hp_multiplier": 1.5},
        {"spawn_time": 28.0, "monster_id": "monster_02", "lane": 1, "hp_multiplier": 1.5},
        {"spawn_time": 38.0, "monster_id": "monster_03", "lane": 2, "hp_multiplier": 2.5},
        {"spawn_time": 55.0, "monster_id": "monster_01", "lane": 0, "hp_multiplier": 0.5},
        {"spawn_time": 55.5, "monster_id": "monster_01", "lane": 1, "hp_multiplier": 0.5},
        {"spawn_time": 56.0, "monster_id": "monster_01", "lane": 2, "hp_multiplier": 0.5},
        {"spawn_time": 56.5, "monster_id": "monster_01", "lane": 3, "hp_multiplier": 0.5},
        {"spawn_time": 57.0, "monster_id": "monster_01", "lane": 4, "hp_multiplier": 0.5}
      ]
    },
    "stage_002": {
      "map_id": 2,
      "waves": [
        {"spawn_time": 0.0, "monster_id": "monster_01", "lane": 2, "hp_multiplier": 1.2}
      ]
    }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `stages` | Object | 顶层容器，key 为 `stage_id`（如 `stage_001`） |
| `map_id` | int | 关联 `battleMap_config.json` 的 `levels` key（1~5），决定背景+地块 |
| `waves` | Array | 波次列表，按 `spawn_time` 升序排列 |
| `waves[].spawn_time` | float | 从关卡开始计时，**相对秒数**。支持小数（如上例 55.5s） |
| `waves[].monster_id` | String | 必须与 `monster_templates.json` 中的 key 完全一致 |
| `waves[].lane` | int | 怪物出生列 (0=最左, 2=正中, 4=最右) |
| `waves[].hp_multiplier` | float | 血量倍率：`最终HP = base_hp x hp_multiplier` |

> **本模板对应你设计的 5 阶段节奏**:
> - 0~12s: 序章上手 (monster_01, hp x1.0)
> - 20~28s: 精英试炼 (monster_02, hp x1.5)
> - 38s: BOSS 降临 (monster_03, hp x2.5)
> - 55~57s: 终幕割草 (monster_01 x5 只, hp x0.5, 密集刷新)
>
> 阶段 3（完善阵型 28~38s）留白，玩家自由部署。

### 2.2 命名规范速查表

| 资源类型 | 命名模板 | 示例 | 说明 |
|----------|----------|------|------|
| 怪物移动帧 | `monster{mid}_move_{idx}.png` | `monster01_move_00.png` | idx: 00~03, 4帧循环 |
| 怪物攻击帧 | `monster{mid}_attack_{idx}.png` | `monster01_attack_02.png` | 第2帧触发伤害判定 |
| 怪物死亡帧 | `monster{mid}_deal_{idx}.png` | `monster01_deal_03.png` | 4帧播放后 fadeout |
| 地块图 | `gezi_{n}.png` | `gezi_13.png` | n: 01~25, 从左到右从上到下 |
| 背景图 | `ui_BattleBG_pass_{n}.png` | `ui_BattleBG_pass_03.png` | n: 01~05, 1080x2160 全屏 |

> **关键规则**: `{mid}` 不带前导零（`1` 而非 `01`），`{idx}` 带前导零（`00`），`{n}` 带前导零（`01`）。

### 2.3 怪物帧路径解析逻辑

`Enemy.gd` 的 `_load_frames()` 使用的拼接规则 ([Enemy.gd:41-46](Scripts/Entities/Enemy.gd#L41-L46))：

```gdscript
var base := "res://Assets/Monsters/monster_%s/" % monster_id  # monster_id = "01"
# -> "res://Assets/Monsters/monster_01/monster01_move_00.png"
```

因此 `monster_id` 在 JSON 中为 `"monster_01"`，传给 Enemy 时会被 `trim_prefix("monster_")` 截为 `"01"`。

## 三、核心变量对应关系

### 3.1 JSON <-> GDScript 变量映射

| JSON 键名 | 代码位置 | 变量/属性 | 类型 | 说明 |
|-----------|----------|-----------|------|------|
| `stages` | `DataManager.stage_config` | `stage_config["stages"]` | Dictionary | Autoload 启动时加载 |
| `stages.{stage_id}` | `BattleTest.current_stage_id` | `@export var current_stage_id` | String | 必须完全一致，如 `"stage_001"` |
| `stages.{stage_id}.map_id` | `BattleTest._current_map_id` | `_current_map_id` | int | 传给 `GridManager.load_battle_map()` |
| `waves[].monster_id` | `monster_templates.{key}` | 模板查找 key | String | 必须完全一致，如 `"monster_01"` |
| `waves[].lane` | `WaveDirector._spawn()` | `Vector2i(lane, 0)` | int | 0~4, 超出会被 clamp |
| `waves[].hp_multiplier` | `WaveDirector._spawn()` | `base_hp x hp_multiplier` | float | 最终 HP = `int(base_hp * hp_mult)` |
| `monster_templates.{id}` | `WaveDirector._spawn()` | `template.get("base_*")` | Dictionary | base_hp/speed/wall_damage/attack_damage |
| `levels.{map_id}` | `GridManager._load_map_config()` | `levels.get(str(map_id))` | Dictionary | key 是字符串数字 `"1"` |

### 3.2 信号链路

```

  └─ all_spawned==true && _alive_enemy_count==0
       └─ GameEvents.stage_victory.emit(stage_id)

```

| 信号 | 发射方 | 参数 | 监听方 |
|------|--------|------|--------|
| `stage_victory` | `WaveDirector` -> `GameEvents` | `stage_id: String` | `BattleTest._on_stage_victory` |

### 3.3 关键 ID 字符串对照

| 概念 | JSON 中 | 代码中 | 一致性要求 |
|------|---------|--------|------------|

## 四、编辑器操作步骤 -- Tile_1 ~ Tile_25

### 4.1 节点结构

在 `Scenes/UI/BattleUI.tscn` 中，节点树应为：

```
BattleUI (CanvasLayer)
├── Wall (TextureRect)
│   ├── HPBarBg (ColorRect)
│   ├── HPBarFill (ColorRect)
│   └── HPLabel (Label)
├── GridAnchor (Control)
│   ├── Tile_01 (Sprite2D)
│   ├── Tile_02 (Sprite2D)
│   ├── ...
│   └── Tile_25 (Sprite2D)
├── CardTrayAnchor (ColorRect)
│   └── CardStartMarker (Control)
└── HandContainer (Control)
```

### 4.2 Tile 节点创建步骤

1. **选中 GridAnchor** -> 右键 -> Add Child Node -> 选择 `Sprite2D`，命名为 `Tile_01`
2. **设置 Texture**: 在 Inspector 中拖入任意占位图（会被代码覆盖）
3. **关键设置 -- Mouse Filter**:
   - 选中 Tile_01 -> Inspector -> `Mouse` -> `Filter` -> 设为 **Ignore**
   - 原因: 地块不响应点击；如果设为 Pass/Stop，会阻断拖拽卡牌时 `get_global_mouse_position()` 的穿透检测
   - 对所有 Tile_01 ~ Tile_25 重复此操作

   > 使用 `Ctrl+D` (Duplicate) 快速复制: 选中 Tile_01 -> `Ctrl+D` 24 次 -> 批量重命名 -> 批量设置 Mouse Filter = Ignore

4. **位置规则**: 代码 `_position_all_tiles()` 会根据 `_cell_size` 和 `_grid_center` 自动计算每个 Tile 的世界坐标，**无需手动摆放**。Tile 节点在编辑器中停留在 (0, 0) 即可。

   - Tile_01 = 第0行第0列 (grid_pos 0,0)
   - Tile_02 = 第0行第1列 (grid_pos 1,0)
   - ...
   - Tile_25 = 第4行第4列 (grid_pos 4,4)

   即 `index = row * 5 + col + 1`，先从左到右，再从上到下。

### 4.3 其他节点 Mouse Filter 设置

| 节点 | Mouse Filter | 原因 |
|------|-------------|------|
| Tile_01~25 | **Ignore** | 不阻挡卡牌拖拽 |
| GridAnchor | **Ignore** | 仅作逻辑容器 |
| CardTrayAnchor | **Pass** | 手牌托盘内的卡牌需要响应拖拽 |
| Wall | **Ignore** | 城墙不响应点击 |
