《掌上英雄 V1.0》项目开发大纲
1. 项目综述与核心概念
1.1 游戏基本信息
1.1.1 游戏定位：竖屏 Roguelike 塔防卡牌
1.1.2 目标平台：移动端（Portrait 模式）
1.1.3 技术选型：Godot 4.x (Compatibility 渲染后端)
1.2 核心循环 (Core Loop)
1.2.1 局内循环：抽卡 -> 部署/合成 -> 关卡推进 -> 随机奖励
1.2.2 局外循环：副本结算获取碎片与金币 -> 英雄等级提升与城墙养成 -> 挑战高难副本
2. 战斗系统逻辑 (Battle System)
2.1 战场布局与基础交互
2.1.1 5x5 棋盘格点坐标映射与动态背景融合
2.1.2 角色拖拽反馈：实时跟随、缩放特效与倾斜动画
2.1.3 强制位移机制：格子占用判定与怪物向上挤压逻辑
2.2 角色战斗行为
2.2.1 纵向攻击机制：RayCast2D/ShapeCast2D 的检测应用
2.2.2 拦截系统：全英雄通用物理阻挡机制与攻击状态切换
2.2.3 英雄属性动态计算：基础值、战中合成加成、特定卡牌加成
2.3 怪物行为与城墙交互
2.3.1 怪物属性体系：移动、碰撞伤害、城墙破坏值、血量缩放
2.3.2 城墙受击反馈：HP 扣除逻辑与屏幕震动 (Screen Shake) 信号
2.3.3 死亡动画：基于 Tween 的缩放消散与全局信号触发
3. 卡牌系统与手牌管理 (Card & Hand Management)
3.1 卡牌流转机制
3.1.1 隐藏备用队列 (Hidden Queue) 的存储与自动填充逻辑
3.1.2 手牌槽位管理：固定上限、位置对齐与 9+ 数量提示
3.1.3 弃牌逻辑：[X] 按钮的交互判定与单次销毁流程
3.2 死锁救援机制 (Anti-Lock System)
3.2.1 触发判定：场上单位与手牌数量双零检测
3.2.2 冷却控制：10 秒强制 CD 与触发间隔锁定
3.2.3 补给包内容：基础角色卡强制生成策略
3.3 卡牌分类与功能定义
3.3.1 角色卡：ID 标识、品质底版、上阵逻辑
3.3.2 效果提升卡：特定角色双路线（攻速/爆发）分支选择逻辑
3.3.3 全局与传说卡：范围 Buff、全场时空停滞等特技实现
4. 战中合成与分支进化 (In-Battle Progression)
4.1 合成升级逻辑
4.1.1 1-5 星级提升的数值映射
4.1.2 视觉进化：体型放大倍率、光环亮度与流光特效控制
4.2 战斗分支路径 (Implicit Branching)
4.2.1 属性加成锁定：首次投喂确定英雄流派逻辑
4.2.2 路线 A (攻速/防御) 的五星质变效果定义
4.2.3 路线 B (爆发/反伤) 的五星质变效果定义
5. 局外养成系统 (Meta-Progression)
5.1 英雄 30 级养成线
5.1.1 碎片与金币消耗公式表设计
5.1.2 5 级间隔质变节点：功能说明、图标显示与解锁判定
5.1.3 英雄基础属性（ATK/HP/DEF）的持久化增长
5.2 城墙养成体系
5.2.1 线性等级提升逻辑与 HP 加成系数
5.2.2 金币消耗公式：基础值与等级指数缩放计算
5.3 奖励与掉落逻辑
5.3.1 结算奖励：金币固定收益与碎片随机概率分配
5.3.2 星级挑战目标：通关、血量占比的任务判定
6. UI/UX 界面与交互 (Interface Design)
6.1 主框架与导航
6.1.1 五 Tab 底部导航：商城、英雄、战斗、营地、公会入口逻辑
6.1.2 英雄界面：槽位拖拽上阵、列表显示与品质底版切换
6.1.3 战斗首页：Boss 预览切换、星级目标预览与难度标识
6.2 弹窗与交互细节
6.2.1 80% 覆盖率半透明弹窗的通用基类与缩放动画
6.2.2 栈式弹窗管理：顶层关闭逻辑与屏幕点击遮罩
6.2.3 结算界面流程：奖励图标的依次弹出与 Tween 曲线应用
6.3 战斗反馈 UI
6.3.1 伤害飘字系统：普通/暴击颜色区分与位置漂移逻辑
6.3.2 血条显示：精英/Boss 怪物的 TextureProgressBar 动态更新
7. 数值表格与技术架构 (Technical Design)
7.1 全表驱动配置 (Resource-Based)
7.1.1 关卡配置表：难度系数、背景图、棋盘贴图关联
7.1.2 角色成长表：等级消耗、属性收益、特殊效果定义
7.1.3 错误码表：ID 映射、多语言文案与 Toast 触发机制
7.2 程序底层架构
7.2.1 全局信号总线 (Event Bus)：EventBus.gd 信号定义汇总
7.2.2 数据持久化：ConfigFile 加密存储方案
7.2.3 场景管理：主界面与战斗场景的异步加载与切换
